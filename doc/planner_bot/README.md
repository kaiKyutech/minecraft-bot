# planner_bot ドキュメント案内

このフォルダのドキュメントが増えてきたため、用途別の読み方ガイドをまとめます。迷ったらここを起点にしてください。

## まず読む
- `API.md`  
  実際のコマンド/呼び出し方/返却フォーマット。外部から使うときの参照先。
- `ARCHITECTURE.md`  
  全体構成と設計思想。仕組みを掴みたいとき。

## 状況や計画を知りたい
- `IMPLEMENTATION_STATUS.md`  
  実装状況のチェックリスト。完了/未実装/予定が一目で分かる。
- `ISSUES.md`  
  現在の課題や既知の問題のメモ。優先度付き。
- `TODO.md`  
  「必ずやる」タスクのリスト。優先度高。

## 振る舞い・設計詳細
- `CHAT_SYSTEM.md`  
  会話/履歴/whisper処理の設計と仕様。
- `paper_idea.md`  
  実験的な構想メモ。実装ソースとは乖離している場合あり。

## メンテナンスの目安（2025-10-時点）
- `ARCHITECTURE.md`, `API.md`, `CHAT_SYSTEM.md`: 最新実装と概ね整合。新機能追加時はここから更新を検討。
- `IMPLEMENTATION_STATUS.md`, `ISSUES.md`, `TODO.md`: 日付が古い項目は現状とズレがある可能性大。内容を確認・整理してから参照推奨。
- `paper_idea.md`: アイデア置き場。古い/未検証の内容を含むため、実装前に現状コードと照合が必要。
